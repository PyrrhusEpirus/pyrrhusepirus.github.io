Этапы процесса создания продукта

Этап предварительной аналитики, направленный на формирование продуктовой гипотезы и подтверждение ее жизнеспособности.

Включает в себя определение:
  • объемов рынков
  • целевой аудитории
  • проблематики
  • проведение исследований (интервью, опросы, статистика и открытые данные)
  • формирование гипотезы

По результатам этапа принимается решение о пилотировании/разработке продукта на основе проверенной ценности для сегмента бизнеса.
⚠Совет! Для выполнения данного этапа можно обратиться в ЦКИ

Описание этапа в Регламенте Р3.1
Возможные виды пилотов:
Пилот (Качественный)
Что происходит на данном этапе:
  • Проверка бизнес-гипотез продуктовых инициатив, тестирование технологических решений, может быть реализован прототип или
    MVP
Ограничения:
  • не предъявляются требования по окупаемости
  • ограничен срок (до 1 года)
  • стоимость не более 50 млн
Результат:
  • подтверждение или опровержение бизнес-гипотез, заявленных целей пилота
  • решение о дальнейших доработке/масштабировании/закрытии продукта
Пилот (Коммерческий)
Выполняется:
  • проверка бизнес-гипотез продуктовых инициатив
  • тестирование технологических решений
  • в рамках пилота может быть реализован
    MVP
Ограничения:
  • предъявляются требования по окупаемости
  • срок не ограничен
  • стоимость не ограничена
Результат:
  • подтверждение или опровержение бизнес-гипотез, заявленных целей пилота
  • решение о дальнейших разработке/масштабировании/закрытии продукта
  • достижение финансовых
    КПЭ

Описание этапа в Регламенте Р3.1
Выполняются разработка архитектуры продукта, оценка стоимости и сроков разработки продукта

Описание этапа в Регламенте Р3.1
Этап, в рамках которого происходит:
  • создание продукта на основе архитектуры продукта (разработка ИТ-решения, подготовка сетевой инфраструктуры)
  • выполнение задач, необходимых для осуществления бизнес-процессов продаж, обслуживания, биллинга нового продукта
  • разработка маркетинговых мероприятий

Описание этапа в Регламенте Р3.1
Эксплуатация продукта в тестовом режиме,
Цель – принятие решения о его эффективности и вводе в коммерческую эксплуатацию

Может быть организована на ограниченное время – до 1 года, на ограниченной территории обслуживания Общества или на ограниченные сегменты клиентов

ОКЭ
может осуществляться с ограничением по количеству клиентов и/или по географическому признаку

Описание этапа в Регламенте Р3.1
На этапе осуществляется полноценная тарификация услуг

Без ограничения по количеству клиентов и по географическому признаку

Не может осуществляться без утвержденного плана продаж и продвижения продукта

В рамках
КЭ
может выполняться развитие

Описание этапа в Регламенте Р3.1
Выход на этот этап может происходить с любого предыдущего этапа, если принято решение о нецелесообразности дальнейшего развития продукта

Включает в себя комплекс мероприятий, направленных на завершение эксплуатации продукта и исключение продукта из портфеля продуктов Общества
